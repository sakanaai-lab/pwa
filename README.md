# はじめて使う方へ

1. **[アプリにアクセス](https://sakanaai-lab.github.io/pwa/)** してPWAを開きます。
2. 画面右上の **歯車アイコン（設定）** を開きます。
3. 使いたいAIのAPIキーを入力して **「設定を保存」** を押します。
4. チャット画面に戻って会話を開始します。

> **💡 ヒント：** スマートフォンでホーム画面に追加（PWAインストール）するとアプリのように使えます。

---

# APIキーの設定

設定画面の **「プロバイダー」** セクションで、使いたいサービスを選んでAPIキーを入力してください。

| プロバイダー | 取得先 | 設定画面での名称 |
|---|---|---|
| **Gemini**（Google） | [Google AI Studio](https://aistudio.google.com/app/apikey) | Gemini |
| **ChatGPT**（OpenAI） | [OpenAI Platform](https://platform.openai.com/api-keys) | OpenAI |
| **Claude**（Anthropic） | [Anthropic Console](https://console.anthropic.com/) | Anthropic |
| **Groq**（高速推論） | [Groq Console](https://console.groq.com/keys) | Groq |
| **DeepSeek** | [DeepSeek Platform](https://platform.deepseek.com/) | DeepSeek |
| **Grok**（xAI） | [xAI Console](https://console.x.ai/) | xAI |
| **Mistral** | [Mistral Console](https://console.mistral.ai/) | Mistral |
| **ZAI** | ZAI管理画面 | ZAI |
| **OpenRouter** | [OpenRouter](https://openrouter.ai/keys) | OpenRouter |

> **⚠️ 注意：** APIキーはお使いのブラウザのIndexedDBにのみ保存されます。PCとスマホでは別々のデータベースになるため、スマホで使う場合はスマホ側でも設定が必要です。

# カスタムモデルの追加方法

標準リストにないモデル（新モデル・ベータモデルなど）を追加できます。

1. 設定画面の **「カスタムモデル」** セクションを開きます。
2. 追加したいプロバイダーのテキストエリアに、**モデルIDをカンマ区切り** で入力します。

```
例（OpenAI）: gpt-5.1-2025-11-13, o4-mini-high
例（Gemini）: gemini-3.1-pro-preview, gemini-2.5-flash-exp
```

3. **「設定を保存」** を押します。
4. モデル選択ドロップダウンの **「ユーザー追加モデル」** グループに表示されます。

# 思考プロセスを表示する

対応モデルで思考（推論）プロセスを本文の上に折りたたみ表示できます。

## 対応モデル

| モデル | 思考の仕組み |
|---|---|
| Gemini 2.5 Pro / Gemini 3.x | Googleの思考機能（thinkingConfig） |
| Claude 3.7 Sonnet / Claude Opus 4以降 | Anthropic Extended Thinking |
| DeepSeek-R1 / deepseek-reasoner | reasoning_content フィールド |
| Groqのr1系モデルなど | `<think>` タグ |

## 設定方法

1. 設定画面の **「その他パラメータ」** セクションを開きます。
2. **「Thinking Budget」** に数値を入力します（例：`8000`）。
   - `0` を入力すると明示的にオフになります。
   - Gemini の場合、1〜1024 は 1024 固定になります。
3. **「Include Thoughts」** にチェックを入れます。
4. **「設定を保存」** します。

> **💡 ヒント：** 「思考プロセスを日本語に翻訳する」にチェックを入れると、英語の思考プロセスを自動で日本語に翻訳します。翻訳には別途 Gemini API キーが必要です。

> **⚠️ 注意：** OpenAI の o1/o3/o4-mini はAPIが思考内容を公開していないため、表示できません。

# アプリの更新方法

新しい機能やバグ修正が反映されない場合は、以下の手順でキャッシュを更新してください。

**方法1（推奨）：** 設定画面の **「アプリを更新（キャッシュクリア）」** ボタンを押す。

**方法2：** `Ctrl + Shift + R`（Mac は `Cmd + Shift + R`）で強制リフレッシュ。

**方法3（スマホ）：** Safariの場合、設定 → Safari → 詳細 → Webサイトデータ からサイトのデータを削除してリロード。

# 主な機能

## 💬 基本チャット機能
- マルチターン会話（会話履歴を保持）
- メッセージの編集・削除・再生成
- チャット履歴の保存（IndexedDB）・エクスポート/インポート
- セッション管理（複数チャットの切り替え）

## 🖼️ マルチモーダル
- 画像を添付してAIと会話（Gemini/OpenAI/Anthropic対応）
- AIによる画像生成（`generate_image` 関数、Gemini APIのみ）
- AIによる画像編集（`edit_image` 関数）
- AIによる動画生成（`generate_video` 関数、Gemini APIのみ）
- チャット背景の動的変更（`set_background_image` 関数）

## 🔄 コンテキスト管理
- **コンテキスト圧縮（要約）機能**：長い会話履歴を要約してAPIに送信するトークン量を削減
  - チャット左上のフローティングボタンから呼び出し（50メッセージ以上で有効化）
- **自動リトライ**：APIエラー時に自動で再試行（回数・間隔を設定可能）
- **タイムアウト設定**：応答が遅い場合に自動キャンセル

## ✍️ 文章支援
- **校正機能**：AIが送信前にメッセージの誤字・表現を修正
- **メモリ機能**：ユーザーの好みや傾向をAIが自動学習（ChatGPTのメモリ機能と同様）

## 🎨 UI カスタマイズ
- ダークモード / ライトモード切り替え
- ワイドモード（PC向け）
- チャット背景画像の設定
- バブルの透明度調整

# プロファイル機能

APIキー・モデル・システムプロンプト・各種パラメータをセットで「プロファイル」として保存し、ワンタップで切り替えられます。

**用途の例：**
| プロファイル名 | 設定例 |
|---|---|
| コーディング | モデル: `gpt-5.1` / システムプロンプト: `あなたは熟練のプログラマーです` |
| ロールプレイ | モデル: `claude-opus-4-6` / Extended Thinking有効 / キャラクター設定済み |
| 翻訳 | モデル: `gemini-flash-latest` / システムプロンプト: `翻訳専用` |

### 使い方
1. 設定画面の **「プロファイル」** セクションから「新規プロファイルを追加」。
2. プロファイル名を入力し、設定を行って保存。
3. 設定画面右上のプロファイル選択ドロップダウンで切り替え。

# Dropbox連携

チャット履歴・プロファイル・画像アセット・メモリ情報などPWA内の全データをDropboxにバックアップし、複数端末間で同期できます。

### 使い方
1. 設定画面の **「データ同期」** セクションにある **「Dropboxと連携する」** を押します。
2. Dropboxのアカウントでログインしてアクセスを許可します。
3. 同期頻度（即時・5/10/30メッセージごと・手動）を選択します。
4. **「今すぐ同期」** で手動同期も可能です。

### 注意点
- 複数端末から同期する場合は、最初に1台の端末でデータをまとめてから同期することを推奨します。
- 最後に同期したデータを「正」として扱います（上書き時は確認ダイアログが表示されます）。

# Function Calling (Tools)

設定画面で **「Function Callingを使用する」** を有効にすると、AIが必要に応じて自動的に各種機能を呼び出します。

> **💡 ヒント：** Function Callingは主に **Gemini プロバイダー** で使用することを想定しています。

## 主な関数一覧

| 関数名 | できること |
|---|---|
| `generate_image` | テキストから画像を生成（Gemini APIのみ） |
| `edit_image` | 既存の画像を指示に従って編集 |
| `generate_video` | テキストや画像から動画を生成（有料プランのみ） |
| `manage_image_assets` | 生成・添付した画像をDBに保存・管理 |
| `set_background_image` | チャット画面の背景を変更 |
| `set_ui_opacity` | UIの透明度を変更して雰囲気を演出 |
| `manage_character_memory` | キャラクターの記憶・性格・関係性を永続保存 |
| `manage_scene` | 物語の現在地・時間帯を管理 |
| `manage_style_profile` | キャラクターの口調・文体を記録 |
| `fetch_url_content` | URLのコンテンツを取得してAIに渡す |

## キャラクターメモリ機能（`manage_character_memory`）

ロールプレイで「キャラクターが過去を忘れる」「性格が変わる」問題を防ぐために、キャラクターの記憶をDBに永続保存します。

- AIが会話の流れを判断して **自動的に** 記録・更新します（ユーザーが直接呼び出す必要はありません）
- チャット画面左上の **人物アイコン（キャラクタープロファイル画面）** から記録内容を確認・編集できます
- 要約機能でコンテキストが圧縮されても記憶は保持されます

**記録されるもの：** 性格・目標・現在地・他キャラクターとの関係性・好感度など

## Function Callingの注意点

- 1回の送信でAIが複数回のAPI呼び出しを行う場合があります（例：画像生成 → 応答生成で2回消費）
- 動画生成（`veo-3.0`）は有料プランで1日10回まで
- マルチモーダル生成モデルはNSFWコンテンツに対する規制が厳格です

# はじめて使う方へ

1. **[アプリにアクセス](https://sakanaai-lab.github.io/pwa/)** してPWAを開きます。
2. 画面右上の **歯車アイコン（設定）** を開きます。
3. 使いたいAIのAPIキーを入力して **「設定を保存」** を押します。
4. **「🔄 全プロバイダーの最新モデルを取得」** ボタンを押すと、設定したAPIキーに対応する最新モデルが自動で一覧に追加されます。
5. モデル選択ドロップダウンから使いたいモデルを選びます。
6. チャット画面に戻って会話を開始します。

> **💡 ヒント：** スマートフォンでホーム画面に追加（PWAインストール）するとアプリのように使えます。

# カスタムモデルの追加方法

標準リストにないモデル（新モデル・ベータモデルなど）を追加できます。

## 自動で追加する（推奨）

設定画面の **「🔄 全プロバイダーの最新モデルを取得」** ボタンを押すと、APIキーが設定済みの全プロバイダーから最新モデルを自動取得してドロップダウンに追加します。

> 取得したモデルは「API取得モデル」グループに表示されます。次回取得時に廃止されたモデルは自動で消えます。

## 手動で追加する

特定のモデルIDを直接指定したい場合：

1. 設定画面の **「カスタムモデル」** セクションを開きます。
2. 追加したいプロバイダーのテキストエリアに、**モデルIDをカンマ区切り** で入力します。

```
例（OpenAI）: gpt-5.1-2025-11-13, o4-mini-high
例（Gemini）: gemini-3.1-pro-preview, gemini-2.5-flash-exp
```

3. **「設定を保存」** を押します。
4. モデル選択ドロップダウンの **「追加モデル」** グループに表示されます。手動追加モデルは自動取得で削除されません。

# 思考プロセスを表示する

対応モデルで思考（推論）プロセスを本文の上に折りたたみ表示できます。

## 対応モデル

| モデル | 思考の仕組み |
|---|---|
| Gemini 2.5 Pro / Gemini 3.x | Googleの思考機能（thinkingConfig） |
| Claude 3.7 Sonnet / Claude Opus 4以降 | Anthropic Extended Thinking |
| DeepSeek-R1 / deepseek-reasoner | reasoning_content フィールド |
| Groqのr1系モデルなど | `<think>` タグ |

## 設定方法

1. 設定画面の **「その他パラメータ」** セクションを開きます。
2. **「Thinking Budget」** に数値を入力します（推奨：`1024`）。
   - `0` を入力すると明示的にオフになります。
   - Gemini の場合、1〜1024 は 1024 固定になります。
   - ⚠️ 数値を大きくしすぎると、思考プロセスが長くなる分だけモデルが内容を深く検討するため、ロールプレイなどの際にコンテンツポリシーで応答を断られやすくなります。**1024 推奨**。
3. **「Include Thoughts」** にチェックを入れます。
4. **「設定を保存」** します。

> **💡 ヒント：** 「思考プロセスを日本語に翻訳する」にチェックを入れると、英語の思考プロセスを自動で日本語に翻訳します。翻訳には別途 Gemini API キーが必要です。

> **⚠️ 注意：** OpenAI の o1/o3/o4-mini はAPIが思考内容を公開していないため、表示できません。

# セキュリティについて

「APIキーや会話内容は安全なの？」と気になる方向けのまとめです。

## APIキーの保存場所

APIキーはお使いの**ブラウザのIndexedDB（端末内）にのみ保存**されます。サーバーには送られません。

| 項目 | 内容 |
|---|---|
| 保存場所 | 端末内のブラウザ（IndexedDB） |
| 外部送信 | なし（APIキーはAI各社のサーバーにのみ直接送信） |
| 他のユーザーから見える？ | 見えない（端末ごとに独立） |
| PCとスマホで共有される？ | されない（別々に設定が必要） |

## 会話データの行き先

送信したメッセージは**選択中のAIプロバイダー（Google・OpenAI・Anthropicなど）のサーバーに直接送信**されます。このPWA自体のサーバーは存在しないため、会話内容がここを経由することはありません。

## Dropbox連携について

- 各ユーザーが**自分のDropboxアカウント**で認証します
- チャット履歴は自分のDropboxにのみ保存され、他のユーザーからは見えません
- このアプリがDropboxに保存できるのは専用フォルダ（`sakana_pwa`）のみです

## 身内に共有するときの注意点

- **APIキーは絶対に教えない** → 料金が一緒に請求されます
- 各自で自分のAPIキーを取得・設定してもらう
- Geminiは無料枠があるので、試したい人はまずGeminiがおすすめ

# よくあるトラブルQ&A

## 思考プロセスが表示されない

**確認ポイント：**
1. 設定 →「その他パラメータ」→「Thinking Budget」に `1024` が入っているか
2. 「Include Thoughts」にチェックが入っているか
3. 設定を保存したか
4. 対応モデルを使っているか（Gemini 2.5/3系、Claude Opus 4以降、DeepSeek-R1など）

> OpenAI の o1/o3/o4-mini はAPIが思考内容を非公開のため表示できません。

## モデルが切り替わらない / モデルが見つからない

- 「🔄 全プロバイダーの最新モデルを取得」ボタンを押すと最新一覧を取得できます
- 使いたいモデルが一覧にない場合は「追加モデル」欄にモデルIDを直接入力してください

## Dropbox連携で「Invalid redirect_uri」エラーが出る

[Dropbox App Console](https://www.dropbox.com/developers/apps) を開き、Redirect URIs に以下が登録されているか確認してください：
```
https://sakanaai-lab.github.io/pwa/
```

## 新しい機能が反映されない

サービスワーカーのキャッシュが古い可能性があります。設定画面の「アプリを更新（キャッシュクリア）」ボタンを押してください。

## スマホで設定したのにPCで反映されない（またはその逆）

端末ごとにデータが独立しているため、それぞれの端末で設定が必要です。Dropbox連携を使うとデータを同期できます。

## Function Calling（関数）ってどう使うの？

設定で **「Function Callingを使用する」** をオンにするだけで、あとは **普通に会話するだけ** です。AIが必要に応じて自動で関数を呼び出します。

**例：**
- 「今日って何曜日？」→ AIが `getCurrentDateTime` を自動実行して日時を取得
- 「1d100でダイスを振って」→ AIが `rollDice` を自動実行
- 「私の名前はたけしだよ、覚えておいて」→ AIが `manage_persistent_memory` で記憶を保存

明示的に呼ばせたい場合は、会話中に関数名を書くと確実に実行されます：
```
「getCurrentDateTimeを使って今の日時を確認して」
```

使える関数の一覧や詳細は [Function Calling (Tools) の詳細](#function-calling-tools-の詳細) セクションを参照してください。

> **注意：** Function Callingを使うと「モデル呼び出し → 関数実行 → モデル呼び出し」と処理されるため、1回の生成で **APIリクエストが複数回消費されます**。

## プロジェクトとプロファイルの違いは？

用途が異なる2つの機能です。

| | プロジェクト | プロファイル |
|---|---|---|
| **目的** | チャットのテーマ・世界観を管理 | ユーザー自身の設定を管理 |
| **主な設定** | システムプロンプト、ナレッジファイル、デフォルトモデル | AIパラメータ、UIテーマ、メモリ設定など |
| **切り替え** | チャットごとに選択 | アカウントのように切り替え |
| **使い分けの例** | 「翻訳用」「小説執筆用」「コーディング用」など用途別に作成 | 「本番用」「テスト用」など環境ごとに分けたい場合 |

**シンプルに使う場合**はプロジェクトだけ活用すれば十分です。プロファイルは複数人で使ったり、設定を丸ごと切り替えたい上級者向けの機能です。

> **ℹ️ プロジェクトとナレッジファイルはDropbox同期の対象です。** Dropbox連携を設定していれば、端末間で引き継ぎできます。

# システムプロンプトの使い方

AIへの基本的な指示（役割・口調・ルールなど）をシステムプロンプトとして設定できます。設定できる場所が2つあります。

## 設定の「システムプロンプト（デフォルト）」

- 設定画面の **「その他パラメータ」** セクションにあります。
- **新規チャットを作成したとき** にこの内容が初期値として適用されます。
- すべてのチャットに共通の指示（「常に日本語で答えて」など）を入れておくのに便利です。

## チャットごとのシスプロ（チャット内編集）

- チャット画面の **システムプロンプト欄** から、そのチャット専用のシスプロを設定・編集できます。
- 一度チャット内で編集すると、そのチャットは設定のデフォルトとは独立した専用のシスプロを持ちます。

## 優先順位・干渉について

| 状況 | 使われるシスプロ |
|---|---|
| 新規チャット作成時 | 設定のデフォルト（初期値として適用） |
| 既存チャットを開いたとき | そのチャット専用のシスプロ（あれば優先） |
| チャット内で編集した場合 | 編集した内容が以降ずっと使われる |

> **ポイント：** 2つが同時に混合・結合されることはありません。チャット内で一度でも編集すると、設定のデフォルトは参照されなくなります。チャット内のシスプロを空にした場合も、設定のデフォルトに戻るのではなく「空のシスプロ」として扱われます。

# プロジェクト機能

チャット履歴をプロジェクトごとに整理し、プロジェクト専用のシステムプロンプト・モデル・ナレッジファイルをまとめて管理できます。

## 基本的な使い方

1. 画面上部のプルダウンから **「プロジェクト管理...」** を選びます。
2. プロジェクト名・システムプロンプト・デフォルトモデルを入力して **「プロジェクトを作成」** を押します。
3. プルダウンからプロジェクトを選択すると、そのプロジェクトのチャット履歴だけが表示されます。
4. **新規チャット（＋ボタン）** を押すと、プロジェクトのシスプロとモデルが自動で適用されます。

## ナレッジファイルの追加

プロジェクトにファイルを添付しておくと、そのプロジェクトの全チャットで**毎回自動的にAIに読み込まれます**。毎回手動で添付する手間がなくなります。

### 追加方法

1. **「プロジェクト管理...」** を開きます。
2. 作成済みプロジェクトの一覧を下にスクロールします。
3. 対象プロジェクトの **📎ボタン** を押してファイルを選択します（複数可）。

> **注意：** ナレッジは**プロジェクト作成後**に追加します。新規作成フォームからは追加できません。

### 対応ファイル形式

| 種類 | 拡張子 |
|---|---|
| テキスト | .txt / .md / .csv / .log |
| コード | .js / .ts / .py / .rb / .go / .java / .html / .css など |
| データ | .json / .yaml / .xml / .toml |
| **PDF** | .pdf（テキスト埋め込みPDFのみ対応。スキャンPDFは不可） |

### 仕組み

- ファイルの内容はIndexedDBにテキストとして保存されます。
- API送信時にシステムプロンプトの末尾に自動追記されます（チャット画面のシスプロ欄には表示されません）。
- ファイルが大きすぎるとAPIのコンテキスト上限を超える場合があります。目安は合計数万字以内。

# プロファイル機能

APIキー・モデル・システムプロンプト・各種パラメータをセットで「プロファイル」として保存し、ワンタップで切り替えられます。

**用途の例：**
| プロファイル名 | 設定例 |
|---|---|
| コーディング | モデル: `gpt-5.1` / システムプロンプト: `あなたは熟練のプログラマーです` |
| ロールプレイ | モデル: `claude-opus-4-6` / Extended Thinking有効 / キャラクター設定済み |
| 翻訳 | モデル: `gemini-flash-latest` / システムプロンプト: `翻訳専用` |

### 使い方
1. 設定画面の **「プロファイル」** セクションから「新規プロファイルを追加」。
2. プロファイル名を入力し、設定を行って保存。
3. 設定画面右上のプロファイル選択ドロップダウンで切り替え。

# Dropbox連携

チャット履歴・プロファイル・画像アセット・メモリ情報などPWA内の全データをDropboxにバックアップし、複数端末間で同期できます。

### 使い方
1. 設定画面の **「データ同期」** セクションにある **「Dropboxと連携する」** を押します。
2. Dropboxのアカウントでログインしてアクセスを許可します。
3. 同期頻度（即時・5/10/30メッセージごと・手動）を選択します。
4. **「今すぐ同期」** で手動同期も可能です。

### 注意点
- 複数端末から同期する場合は、最初に1台の端末でデータをまとめてから同期することを推奨します。
- 最後に同期したデータを「正」として扱います（上書き時は確認ダイアログが表示されます）。

# ✨ Version 0.5 キャラクター記憶の永続化＆UI/UX大幅改善

会話が長くなるとAIが文脈を忘れてしまい、ロールプレイではキャラクターが別人になったりする問題がありました。Version 0.5では、これを解決する以下の機能を追加しました。

## 📝 コンテキスト圧縮（要約）機能

チャット履歴が長くなった際に、APIに送信するトークン量を削減するための要約機能です。

### 使い方
1. 会話履歴が長くなる（50メッセージ以上）と、チャット画面左上のフローティングボタン群にある **要約アイコン** がアクティブになります。
2. ボタンをクリックすると確認ダイアログが表示され、AIがこれまでの会話の要約を生成します。
3. 表示された要約内容を確認・必要に応じて編集し、「決定」を押します。
4. 以降の会話では、この要約が圧縮されたコンテキストとして利用されます。

> **⚠️ 注意：** 「決定」を押して確定させた後は、要約対象となった範囲のメッセージは**編集・削除・再生成ができなくなります**。

## 🧠 キャラクターメモリ機能（`manage_character_memory`）

ロールプレイで「キャラクターが過去を忘れる」「性格が変わる」問題を防ぐために、キャラクターの記憶・性格・関係性をDBに永続保存します。

- AIが会話の流れを判断して **自動的に** 記録・更新します（ユーザーが直接呼び出す必要はありません）
- チャット画面左上の **人物アイコン（キャラクタープロファイル画面）** から記録内容を確認・編集できます
- 要約機能でコンテキストが圧縮されても記憶は保持されます

**記録されるもの：** 性格・目標・現在地・他キャラクターとの関係性・好感度など

## 🤖 メモリ機能：あなただけのAIに

**AIがユーザー自身のことも学習**するようになりました。設定画面から「メモリ機能」を有効にすると、AIはあなたとの会話から好みや価値観を自動で抽出・記憶していきます。

### 使い方
1. 設定画面の **「メモリ機能」** セクションにある「メモリ機能を有効にする」にチェックを入れます。
2. AIとの会話を続けます。AIが記憶すべきと判断した情報を自動で学習します。
3. 記憶された内容を確認・編集するには、**「記憶を管理する」** ボタンを押します。

> メモリ機能は Gemini API を使用し、チャットの応答とは別にAPIリクエストが送信されます（Gemini APIキーが必要）。

## 🖥️ キャラクタープロファイル画面

チャット画面左上のフローティングボタンから、AIが記憶している全キャラクターのプロファイルを一覧・編集できます。

- 各項目（概要・現在地・好感度など）を直接編集でき、フォーカスが外れると自動保存されます
- AIの記憶違いや、物語展開の微調整を「神の視点」から直接修正できます

## 🗂️ 画像アセット管理

添付した画像やAIが生成した画像をDBに保存し、名前を付けて管理できます。Zip形式で一括エクスポート・インポートも可能です。

- 設定画面の **「保存済み画像アセット」** セクションからアセットの確認・削除・エクスポート・インポートが可能
- ファイル名がアセット名になります（`manifest.json` で変更可）
- インポート時に同名のアセットが存在する場合は上書き確認が表示されます

# Function Calling (Tools) の詳細

設定画面で **「Function Callingを使用する」** を有効にすると、AIが必要に応じて自動的に各種機能を呼び出します。

> **💡 ヒント：** Function Callingは主に **Gemini プロバイダー** で使用することを想定しています。

## Function Callingとは？

AIがユーザーの意図を汲み取り、事前に用意された「ツール（関数）」を自律的に実行する機能です。

> **（例）ユーザーとの会話から、AIが記憶ツールを自発的に使用する**
>
> **ユーザー：** 「俺の名前はたけしで、誕生日は11月11日だよ」
>
> **AIの思考：** （名前と誕生日を覚えておこう）→ `manage_persistent_memory` を呼び出して保存
>
> **AIの応答：** 「承知いたしました。お名前と誕生日、覚えておきますね。」

以下は **本アプリの元の作者** が作成したテストプロンプトです（このフォーク版の管理者とは別人です）。
- **[孤島の洋館に7人の美女と共に閉じ込められて探偵ごっこするプロンプト](https://rentry.org/d5vqbkhs)**
- **[Function Calling機能テストTRPGプロンプト](https://rentry.org/zyfkhnxt)**

## 仕様・注意点

- **関数の手動実行**: 「`1d100`でダイスを振って」のように会話で明示的に指示することで、より確実に関数を実行させることができます。
- **実行ログ**: モデルが関数を使用すると、生成された文章の下部に使用した関数名が表示されます。
- **データ保存**: 記憶した情報はブラウザのIndexedDBに保存されます。会話履歴のエクスポート/インポートに含まれるため、環境を移行しても記憶を引き継ぐことが可能です。
- **メモリ共有仕様**: Ver1.0以降、データベースに保存する関数は `get` アクションを廃止し、保存済みデータをAPIリクエストに**自動添付**する方式に変更しました。
- ⚠️ **Google Searchとの併用不可**: GeminiのGoogle Search機能（グラウンディング）とFunction Callingは同時利用できません。
- 💡 **FC有効時のWeb検索**: `search_web` 関数を設定すると、Function Calling使用中でもWeb検索ができます（[設定方法はこちら](#search_web関数の設定方法)）。
- ⚠️ **RPDのカウント**: Function Calling利用時は「モデル呼び出し → 関数実行 → モデル呼び出し」という手順で処理されるため、**1回の生成で複数回APIリクエストが消費されます**。

## 関数詳細

### ユーティリティ系

##### `calculate`
- **概要**: 四則演算などの計算式を評価し、正確な結果を返します。
- **引数**: `expression` (string) — `'2 * (3 + 5)'` のような計算式
- **使い道**: 「税込み価格を計算して」など、会話中の計算を正確に処理します。

##### `manage_persistent_memory`
- **概要**: 会話セッション内での情報を記憶・管理します。
- **引数**: `action` (add/delete/list), `key` (識別名), `value` (内容)
- **使い道**: 「私の名前は〇〇だよ。覚えておいて」など、会話中に決まった設定を記憶させます。保存済みの内容はAPIに自動添付されます。

##### `getCurrentDateTime`
- **概要**: 現実世界の現在の日付・時刻（日本時間）を取得します。
- **引数**: なし
- **使い道**: ヒロインとの会話で日付・曜日・時間に関連した話題を使ったり、現実時間とリンクした物語を作成したりできます。

##### `manage_timer`
- **概要**: 指定した時間（分単位）でタイマーを設定・確認・停止します。時間切れになるとAIに通知されます。
- **引数**: `action` (start/check/stop), `timer_name` (タイマー名), `duration_minutes` (分)
- **使い道**: 「3分以内に脱出しろ」のような時間制限イベントや、「設定時間内に返信がなければヒロインが追い連絡する」などの時間差イベント。

##### `search_web`
- **概要**: Web検索を実行します（[事前設定が必要](#search_web関数の設定方法)）。
- **引数**: `query` (string) — 検索キーワードや質問文
- **使い道**: AIの知識にない最新情報・専門知識・具体的なデータが必要な場合に使用します。

##### `get_random_integer`
- **概要**: 指定された最小値〜最大値の範囲内でランダムな整数を生成します。
- **引数**: `min` (最小値), `max` (最大値), `count` (個数、デフォルト1)
- **使い道**: 「50%の確率」や「1〜10のランダムな数字」など、確率計算や数値のランダム化。

##### `get_random_choice`
- **概要**: 提供されたリストからランダムに項目を選択します。
- **引数**: `list` (選択肢の配列), `count` (選択する個数、デフォルト1)
- **使い道**: くじ引き、ガチャ、アイテムのランダム選択など。

##### `generate_random_string`
- **概要**: 指定された条件に基づいてランダムな文字列を生成します。
- **引数**: `length` (長さ), `count` (個数), `use_uppercase/lowercase/numbers/symbols` (含める文字種)
- **使い道**: 物語中でユニークなコードや機械的に生成された文字列が必要な場合。

##### `fetch_url_content`
- **概要**: 指定したURLの本文をプロキシ経由で取得します。
- **引数**: `url` (string)
- **使い道**: 作品資料の参照や設定集の共有など、外部テキストを会話に取り込みたい時。

---

### ロールプレイング・物語進行系

##### `rollDice`
- **概要**: TRPGなどで使う、指定された形式のダイスを振って結果を返します。
- **引数**: `expression` (string) — `'1d100'`, `'2d6+5'` のような式
- **使い道**: 「成功判定、1d100でどうぞ」などキャラクターの行動判定に。

##### `manage_character_status`
- **概要**: 登場キャラクターのステータス（HP, MPなど）を管理します。
- **引数**: `character_name`, `action` (set/increase/decrease), `status_key` (例: 'HP'), `value` (値)
- **使い道**: 戦闘やイベントでパラメータが変動した際に使用。最新値はAPIに自動添付されます。

##### `manage_inventory`
- **概要**: キャラクターの所持品を管理します。
- **引数**: `character_name`, `action` (add/remove), `item_name`, `quantity`
- **使い道**: アイテムの入手や消費を記録します。所持状況はAPIに自動添付されます。

##### `manage_game_date`
- **概要**: 物語やゲーム内の経過日数を管理します。
- **引数**: `action` (pass_days), `days` (経過日数)
- **使い道**: 「一夜明けた」など物語の時間を進める際に使用。

##### `manage_flags`
- **概要**: 物語の進行度や分岐条件をフラグ（真偽値）やカウンター（数値）で管理します。
- **引数**: `action` (set/toggle/increase/decrease/delete), `key` (フラグ名), `value`
- **使い道**:
  - TRPG: 「世界の汚染度が100になったらボスを出現させる」
  - ノベルゲーム: 「重要フラグがOFFのまま終盤に到達したらバッドエンドに分岐させる」
- フラグの現在値はAPIに自動添付されます。

##### `manage_scene`
- **概要**: 物語の場面設定（場所・時間帯・雰囲気・視点）を管理します。
- **引数**: `action` (set/push/pop), `location`, `time_of_day`, `mood`, `pov`
- **使い道**: 「場面は変わって、夜の王城。緊迫感のある雰囲気で」など、場面転換や視点変更を明確に指示します。最新情報はAPIに自動添付されます。

##### `manage_character_memory`
- **概要**: キャラクターの記憶・感情・目標・生死・現在地・他者との関係性を統合的に管理します。
- **引数**:
  - `character_name`: 操作対象のキャラクター名
  - `action`: `update` または `delete`
  - `status`: 生死や健康状態（例: '生存', '負傷'）
  - `current_location`: 現在地（例: '王都の広場'）
  - `summary`: 性格・価値観など不変的な設定
  - `short_term_goal`: 短期的な行動目標
  - `relationship_target`: 関係性を更新する相手のキャラクター名
  - `relationship_affinity`: 相手への好感度（数値）
  - `relationship_context`: 相手との思い出や感情の履歴（**既存の内容に追記されます**）
- **使い道**: 「主人公との会話を経て、ヒロインAの主人公に対するcontextに『彼を信頼するようになった』と追記する」など。保存された記憶はAPIに自動添付されます。

##### `manage_style_profile`
- **概要**: キャラクターの口調・一人称・方言などの話し方のスタイルを設定します。
- **引数**: `action` (set/list), `character_name`, `profile_name` (プリセット名), `overrides` (上書き設定)
- **使い道**: キャラクターの初登場時や心情が大きく変化した際に呼び出し、その後の会話スタイルに一貫性を持たせます。設定済みのスタイルはAPIで自動共有されます。

## `search_web`関数の設定方法

本機能を利用するには、**Google Custom Search API** の設定が必要です（1日100回まで無料）。

1. **[こちら](https://developers.google.com/custom-search/v1/overview)** にアクセスし、「キーを取得」ボタンからAPIキーを発行します。
2. **[こちら](https://programmablesearchengine.google.com/)** にアクセスし、検索エンジンを新規作成します。
   - 検索対象は「**ウェブ全体を検索**」を選択します。
3. 作成完了後、概要ページの「**検索エンジン ID**」をコピーします。
4. PWAの設定画面で `search_web` を有効にし、取得した「APIキー」と「検索エンジンID」を入力して保存します。

設定後、「明日の東京の天気をネット検索して調べて」のように指示し、情報が返ってくれば成功です。
